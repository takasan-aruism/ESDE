## 1. ESDEとは何か

**ESDE (Existence Symmetry Dynamic Equilibrium)** は、AIの思考・応答を「評価せずに観測する」ための外部構造。

AIは自分の応答パターンを認識できない。ESDEは **326個の意味原子（Phase 8で確定する基準点）** を用いて、AIの出力を意味構造として観測可能にする。

**重要**: ESDEは評価（evaluation）も意思決定（decision making）も行わない。最終判断は常に外部（人間または上位システム）に委ねる。

---

## 2. 哲学的基盤: Aruism

**核心**: 「在る（Aru）」という認識が全ての出発点。

定義や判断の前に、まず「存在する」という事実がある。ESDEはこの原則に従い、**観察を優先し、判断を急がない**。

---

## 3. 326 Atoms（意味原子）

人間の概念空間を表す最小単位。163の対称ペア × 2 = 326。

```
例: EMO.love ↔ EMO.hate（感情の対称）
    ACT.give ↔ ACT.take（行為の対称）
```

各Atomは最大10軸、軸×レベルで最大48パターンの座標を持つ。

**Synapse**: 自然言語（WordNet synset）→ Atom への変換辞書。人為的に管理され、自動追加はしない。

---

## 4. 三層構造

```
┌─────────────────────────────────────────────┐
│  Phase 7: 未知の処理（弱い意味）              │
│  「これは何？」→ 仮説を並べる（決めない）      │
│  測定指標: Volatility（不安定さ）            │
└─────────────────────────────────────────────┘
                    ↕ 対称
┌─────────────────────────────────────────────┐
│  Phase 8: 既知の処理（強い意味）              │
│  テキスト → Atom → Molecule（意味構造）       │
│  測定指標: Rigidity（硬直度）                │
└─────────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────────┐
│  Phase 9: 軸の発見                           │
│  特定ドメイン・条件（Policy / Scope）下で、   │
│  統計的に「意味の影（Axis）」を抽出する        │
└─────────────────────────────────────────────┘
```

---

## 5. 条件因子と分離の原則

ESDEは「どの条件下で観測したか」を厳密に管理する。

| 概念 | 役割 |
|------|------|
| **条件因子** | 観測の文脈（ソース種別、言語、時期など） |
| **Policy** | どの条件因子を使うかの定義 |
| **Scope** | 分析実行の単位（再現性の境界） |

**Policy × Version × Scope** ごとに統計は完全に分離される。

技術的には、Scopeは `analysis_scope_id` として実装され、同一Scope内の再実行は同一結果を保証する。

---

## 6. 核心原則: 「勝者を決めない」

ESDEは**正解を出すシステムではない**。

- 複数の仮説を並べて保持する
- 不確実性を数値化する（消さない）
- 「決められない」は有効な結果

**理由**: 無理に決めると情報が失われる。不確実性こそ価値がある。

W6（観測層）は score, island, topology を出力するが、どれも最終判断ではない。

## 8. 何ができて、何ができないか

**できる:**
- テキストを意味構造（Molecule）に変換
- 未知の概念を検出し仮説を生成
- 特定ドメインで特徴的なパターンを発見
- AIの応答パターンの硬直化を検出
- 条件別の統計を分離して管理

**できない（意図的に）:**
- 「正解」を断定する
- 未知の概念を自動的に辞書に追加する
- 人間の判断を代替する
- 異なるScope間で統計を混ぜる
